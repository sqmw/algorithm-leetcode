# LEETCODE 刷题笔记

## 一、刷题原则与规范

### 1. 刷题规则
- 难度换算: 1道难题 = 2道中等题 = 6道简单题
- 每日目标：做到需要收费的时候每天4题及其以下
- 160题附近开始收费，此时已完成所有基础数据结构（除图外）

### 2. 编码规范
- 变量定义域按照C语言使用（更加规范）
- 算法实现需同时保证性能和可读性
- 变量命名规则：
  - `des_XXX`: 表示目标值
  - `rec_f`: 表示递归/迭代函数

### 3. 算法实现标准
- 时间复杂度：需要达到最低
- 空间复杂度：不超过O(n)即可，非必要不降低到O(1)

## 二、核心知识体系

### 1. 数据结构基础
- 排序算法：需要全部掌握
- 图论：包括线性表和二叉树（都是特殊的图）
- 哈希：需要能够手动实现

### 2. 算法实现方式
- 循环实现
- 递归实现
- 重要概念：递归 == 循环 + 栈
  - *递归和循环的空间、时间复杂度一般相同*
  - 递归本质是通过栈和循环实现

### 3. 基础数据结构
1. 线性表
2. 字符串（特殊的线性表）
3. 树
   - 线段树(segment tree)
   - 树状数组(Binary Indexed Tree，BIT)
4. 图

#### 4.1 贪心算法
